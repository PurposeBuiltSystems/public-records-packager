Add-in: Public Records Packager

What it does

Searches your own mailbox for records responsive to a public-records request and builds a numbered, chronological bundle in your OneDrive: exact .eml per message, a printable .html copy, attachments in original format, and a manifest.csv with blank exemption columns.

How to use it

  1. Open any message in Outlook and click Records Packager on the ribbon.
  2. Enter search terms (and optionally a date range and participant), then Find responsive messages. Sign in / consent the first time.
  3. Review the chronological list — uncheck anything out of scope.
  4. Name the bundle folder and click Build records bundle in OneDrive. The bundle lands in OneDrive / Records Requests / <your name>.
  5. Perform redaction and exemption review in your normal tools; record determinations in manifest.csv. Print the .html copies to PDF for release versions.

Bundle contents

  • NNNN_date_subject.eml — exact archival copy (opens in Outlook)
  • NNNN_date_subject.html — printable copy (browser → print to PDF)
  • NNNN_attachments/ — that message's files, original format
  • manifest.csv — index with released_in_full / exemption_notes / statute_cited columns for the records officer
  • README.txt — explains the layout to counsel/requesters

Good to know

  • Search is keyword-based (Outlook full-text search) and capped around 1,000 results — run multiple narrower searches for very large requests.
  • No redaction: bundles contain unredacted records — review before release.
  • Chronological numbering is oldest-first, the conventional production order.

Troubleshooting

  • Bundle failed partway: already-uploaded files remain in the OneDrive folder; rerun into a fresh folder name.
  • A message is missing: try different keywords — search matches words, not concepts; also check the date filter.
  • Button doesn't appear: Microsoft 365 / Exchange Online account required.

Privacy

Records never leave your Microsoft 365 tenant — search runs in your own session and the bundle is written to your own OneDrive. See the Privacy Policy.
